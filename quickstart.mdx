title: "Quickstart: accelerate your first download with Xget"
description: "Make your first accelerated request with Xget in under 5 minutes — from a GitHub clone to configuring npm and pip mirrors globally."
---

This guide walks you through making your first accelerated request with Xget, then setting up your development tools to route through Xget automatically. By the end, you'll have Git, npm, and pip configured to use Xget for faster, more reliable access to upstream resources.

<Warning>
  The public instance at `xget.xi-xu.me` is for evaluation and testing only. For production or availability-sensitive workloads, [deploy your own instance](/deployment/cloudflare-workers).
</Warning>

<Steps>
  <Step title="Make your first accelerated request">
    The simplest way to try Xget is with a `wget` or `curl` download. Replace the upstream domain with `xget.xi-xu.me` and add the platform prefix:

    ```bash
    # Original GitHub URL
    # https://github.com/microsoft/vscode/archive/refs/heads/main.zip

    # Accelerated via Xget (add the gh/ prefix)
    wget https://xget.xi-xu.me/gh/microsoft/vscode/archive/refs/heads/main.zip
    ```

    You can also clone a repository directly:

    ```bash
    # Shallow clone via Xget
    git clone --depth 1 https://xget.xi-xu.me/gh/torvalds/linux.git
    ```

    <Tip>
      Not sure how to convert a URL? Use the **[URL Converter](https://xuc.xi-xu.me)** at `xuc.xi-xu.me` — paste any supported platform URL and get the Xget equivalent instantly.
    </Tip>
  </Step>

  <Step title="Configure Git globally">
    Tell Git to automatically route all GitHub (and other platform) traffic through Xget using URL rewrites:

    ```bash
    # GitHub
    git config --global url."https://xget.xi-xu.me/gh/".insteadOf "https://github.com/"

    # GitLab
    git config --global url."https://xget.xi-xu.me/gl/".insteadOf "https://gitlab.com/"

    # Gitea
    git config --global url."https://xget.xi-xu.me/gitea/".insteadOf "https://gitea.com/"

    # Codeberg
    git config --global url."https://xget.xi-xu.me/codeberg/".insteadOf "https://codeberg.org/"

    # Verify the configuration
    git config --global --get-regexp url
    ```

    After this, running `git clone https://github.com/...` will automatically use Xget — no URL changes needed.
  </Step>

  <Step title="Configure npm to use Xget">
    Point npm at the Xget npm registry mirror:

    ```bash
    # Globally configure npm mirror
    npm config set registry https://xget.xi-xu.me/npm/

    # Verify
    npm config get registry
    ```

    Or configure it per-project with `.npmrc`:

    ```bash .npmrc
    registry=https://xget.xi-xu.me/npm/
    ```
  </Step>

  <Step title="Configure pip to use Xget">
    Point pip at the Xget PyPI mirror:

    ```bash
    # Globally configure pip mirror
    pip config set global.index-url https://xget.xi-xu.me/pypi/simple/
    pip config set global.trusted-host xget.xi-xu.me
    ```

    Test it:

    ```bash
    pip install requests
    ```
  </Step>

  </Step>
</Steps>
