Xget uses a simple prefix-based URL convention. To accelerate any supported URL, replace the platform's domain with your Xget instance domain followed by a short platform prefix. The rest of the path is passed through unchanged to the upstream service â€” no path rewriting occurs beyond stripping the prefix.

## Conversion format

```
https://{instance}/{prefix}/{original-path}
```

For example, the GitHub URL `https://github.com/microsoft/vscode/archive/refs/heads/main.zip` becomes:

```
https://xget.xi-xu.me/gh/microsoft/vscode/archive/refs/heads/main.zip
```

The prefix `gh` identifies GitHub as the target platform. Everything after the prefix maps directly to the upstream path.

<Note>
  For most platforms, the path after the prefix maps directly to the upstream service path â€” only the domain changes. A small number of platforms (Rust Crates and Jenkins) apply path normalization to match their upstream API structure, but the URL format you use remains the same.
</Note>

## Conversion examples

The following examples show four common conversions in full.

### GitHub file download

```bash
# Original
https://github.com/microsoft/vscode/archive/refs/heads/main.zip

# Xget-accelerated (replace domain, add gh prefix)
https://xget.xi-xu.me/gh/microsoft/vscode/archive/refs/heads/main.zip

# Download with wget
wget https://xget.xi-xu.me/gh/microsoft/vscode/archive/refs/heads/main.zip
```

### Hugging Face model file

```bash
# Original
https://huggingface.co/microsoft/DialoGPT-medium/resolve/main/pytorch_model.bin

# Xget-accelerated (replace domain, add hf prefix)
https://xget.xi-xu.me/hf/microsoft/DialoGPT-medium/resolve/main/pytorch_model.bin

# Multi-threaded download with aria2c
aria2c -x 16 -s 16 https://xget.xi-xu.me/hf/microsoft/DialoGPT-medium/resolve/main/pytorch_model.bin
```

### npm package fetch

```bash
# Original
https://registry.npmjs.org/react/-/react-18.2.0.tgz

# Xget-accelerated (replace domain, add npm prefix)
https://xget.xi-xu.me/npm/react/-/react-18.2.0.tgz

# Configure npm globally to use Xget
npm config set registry https://xget.xi-xu.me/npm/
```

### Docker Hub image pull

Xget uses a path-based scheme for container registries â€” use the `cr/docker` prefix directly in your image reference:

```bash
# Pull directly using the Xget cr/docker prefix
docker pull xget.xi-xu.me/cr/docker/library/nginx:latest

# Or for user images
docker pull xget.xi-xu.me/cr/docker/nginx/nginx-unprivileged:latest
```
