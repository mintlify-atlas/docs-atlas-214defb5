Whether you're downloading GitHub releases, pulling container images, installing packages, or calling AI inference APIs, Xget routes your requests through Cloudflare's edge infrastructure — bringing upstream services closer to you and dramatically improving speed and reliability.

## Why Xget?

Xget solves a common problem for developers: slow or unreliable access to essential upstream resources. Instead of fighting timeouts and bandwidth constraints, Xget sits between you and the upstream service — caching responses at the edge, retrying transient failures automatically, and enforcing enterprise-grade security headers on every response.

## How it works

Xget uses a simple URL prefix convention. Replace a platform's domain with your Xget instance domain plus a short platform prefix:

```bash
# Original GitHub URL
https://github.com/microsoft/vscode/archive/refs/heads/main.zip

# Xget-accelerated URL
https://xget.xi-xu.me/gh/microsoft/vscode/archive/refs/heads/main.zip
```

The same pattern applies to npm, PyPI, Docker Hub, OpenAI, and every other supported platform. See the [URL conversion guide](/url-conversion) for the full prefix reference.

## Get started

<Steps>
  <Step title="Use the pre-deployed instance">
    The public instance at `xget.xi-xu.me` is available for evaluation. Point any supported URL at it to test acceleration immediately.
  </Step>
  <Step title="Convert your URLs">
    Use the [URL Converter](https://xuc.xi-xu.me) tool or follow the [URL conversion guide](/url-conversion) to rewrite your download, clone, and registry URLs.
  </Step>
  <Step title="Deploy your own instance">
    For production workloads, deploy your own Xget instance. Follow the [Cloudflare Workers](/deployment/cloudflare-workers) or [self-hosted Docker](/deployment/self-hosted) guide.
  </Step>
  <Step title="Configure your tools">
    Globally configure Git, npm, pip, Cargo, and other tools to route through Xget automatically. See the [platform guides](/platforms/code-repositories) for tool-specific instructions.
  </Step>
</Steps>

<Warning>
  The public instance at `xget.xi-xu.me` is intended for evaluation only. For production or availability-sensitive workloads, deploy your own instance and put it behind authentication or IP allowlists.
</Warning>
